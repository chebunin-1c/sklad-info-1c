# Склад Инфо

Конфигурация на платформе **1С:Предприятие 8.3** для автоматизации складского учета и отработки типовых подходов разработки в 1С.

## О проекте

Проект развивается как отдельная конфигурация с базовыми объектами предметной области склада.  
В репозитории фиксируются изменения структуры конфигурации, форм, реквизитов, модулей и других объектов.

Основной акцент сделан на:
- понятную структуру конфигурации;
- аккуратное развитие объектов метаданных;
- пошаговое расширение функциональности;
- использование git для контроля изменений.

## Функциональное направление

В рамках проекта реализуются и развиваются:
- справочники;
- документы;
- формы;
- перечисления;
- реквизиты объектов;
- модули и обработчики событий;
- запросы;
- регистры;
- прикладная логика, связанная со складским учетом.

## Структура репозитория

- `config/` — выгрузка конфигурации в XML-файлы
- `snapshots/` — резервные снимки конфигурации в формате `.cf`
- `.gitignore` — исключения для git
- `README.md` — описание проекта

## Подход к хранению в git

В репозитории хранится **не файловая база 1С**, а **выгрузка конфигурации в файлы**.

Это позволяет:
- отслеживать изменения по объектам конфигурации;
- вести понятную историю развития проекта;
- хранить структуру конфигурации в удобном для контроля версий виде.

## Обновление конфигурации в репозитории

После изменений в Конфигураторе:

1. Открыть конфигурацию в Конфигураторе
2. Выполнить команду **Конфигурация → Выгрузить файлы конфигурации**
3. Выгрузить файлы в папку `config/`
4. Проверить изменения в git
5. Выполнить commit и push

## Восстановление из репозитория

Для загрузки конфигурации из репозитория:

1. Открыть информационную базу в Конфигураторе
2. Выполнить загрузку конфигурации из файлов
3. Указать папку `config/`
4. При необходимости обновить конфигурацию базы данных

> Конкретные команды и поведение могут зависеть от версии платформы 1С.

## Принципы ведения проекта

- хранить в git именно конфигурацию, а не файл базы;
- вносить изменения небольшими логическими шагами;
- использовать понятные комментарии к коммитам;
- поддерживать актуальную структуру проекта;
- при необходимости сохранять `.cf` как отдельную резервную копию.

## Примеры изменений, которые фиксируются в истории

- добавление новых объектов метаданных;
- создание и доработка форм;
- добавление реквизитов;
- изменение модулей и обработчиков;
- расширение прикладной логики;
- подготовка структуры под дальнейшее развитие проекта.

## Техническая информация

- Платформа: **1С:Предприятие 8.3**
- Формат хранения в репозитории: **выгрузка конфигурации в XML-файлы**
- Дополнительные резервные копии: **`.cf` в папке `snapshots/`**

## Назначение проекта

Проект используется как отдельная практическая конфигурация, в которой последовательно развивается прикладная логика складского учета, структура объектов и подходы к разработке в 1С.
